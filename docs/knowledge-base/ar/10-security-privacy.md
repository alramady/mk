# الأمان والخصوصية

يهدف هذا الدليل إلى توضيح السياسات والإجراءات المتعلقة بأمن المعلومات والخصوصية في "المفتاح الشهري". يجب على جميع الموظفين الالتزام بهذه الإرشادات لضمان حماية بيانات الشركة وعملائنا.

## ما يمكن وما لا يمكن مشاركته

من الضروري فهم طبيعة المعلومات التي نتعامل معها وتصنيفها حسب حساسيتها. يوضح الجدول التالي ما يمكن مشاركته وما لا يمكن مشاركته خارج الشركة.

| نوع المعلومة | أمثلة | سياسة المشاركة |
| :--- | :--- | :--- |
| **معلومات عامة** | أسعار الوحدات، صور الوحدات المتاحة، معلومات عن الشركة | **يمكن مشاركتها** مع العملاء والشركاء. |
| **معلومات العملاء الشخصية** | الأسماء، أرقام الهواتف، البريد الإلكتروني، تفاصيل الهوية | **يمنع منعاً باتاً** مشاركتها خارج المنصة. |
| **معلومات الدفع** | تفاصيل بطاقات مدى، أرقام الحسابات البنكية | **محظور تماماً** مشاركتها أو تخزينها خارج نظام الدفع. |
| **بيانات التشغيل الداخلية** | معدلات الإشغال، تقارير الأداء المالي (KPIs) | **للاستخدام الداخلي فقط**، وتتم مشاركتها بين الإدارات المعنية. |
| **مفاتيح API وبيانات Beds24** | مفاتيح الوصول لواجهات برمجة التطبيقات، بيانات تسجيل الدخول لـ Beds24 | **سرية للغاية**، وتقتصر على المسؤولين التقنيين فقط. |

## حماية بيانات العملاء

حماية بيانات العملاء هي أولوية قصوى. يجب اتباع الإجراءات التالية لضمان أمان معلوماتهم:

1.  **الوصول المحدود**: لا تحاول الوصول إلى بيانات العملاء إلا عند الضرورة القصوى لأداء مهامك الوظيفية.
2.  **عدم التخزين المحلي**: لا تقم بتحميل أو تخزين أي بيانات للعملاء على أجهزة الكمبيوتر الشخصية أو الأجهزة المحمولة.
3.  **التواصل الآمن**: استخدم قنوات الاتصال الرسمية المعتمدة من الشركة عند التواصل مع العملاء.
4.  **الإبلاغ عن الخروقات**: في حال الشك بوجود خرق أمني أو تسريب للبيانات، قم بإبلاغ المسؤول المباشر فوراً.

## مفاتيح API

مفاتيح API (مفاتيح واجهة برمجة التطبيقات) هي بمثابة كلمات مرور للأنظمة. تُستخدم لربط نظامنا مع خدمات أخرى مثل Moyasar. التعامل مع هذه المفاتيح يتطلب حذراً شديداً.

-   **صلاحيات الوصول**: يتم منح صلاحيات الوصول لمفاتيح API من قبل قسم تقنية المعلومات فقط.
-   **يمنع مشاركتها**: لا تشارك مفاتيح API مع أي شخص غير مصرح له، سواء داخل الشركة أو خارجها.
-   **التجديد الدوري**: يتم تجديد مفاتيح API بشكل دوري لزيادة مستوى الأمان.

## بيانات Beds24

Beds24 هو نظام إدارة القنوات الذي نعتمد عليه. يحتوي على معلومات حساسة حول الحجوزات والأسعار والتوفر.

-   **تسجيل الدخول**: بيانات تسجيل الدخول إلى Beds24 مخصصة لفريق العمليات (Ops) والمسؤولين (Admin) فقط.
-   **عدم التغيير المباشر**: لا تقم بإجراء أي تغييرات مباشرة على Beds24 إلا بعد الحصول على موافقة من المسؤول. النظام المحلي هو مصدر الحقيقة للبيانات، وأي تغيير يجب أن يتم من خلاله.
-   **سرية البيانات**: بيانات الحجوزات والعملاء على Beds24 تخضع لنفس سياسات الخصوصية المذكورة أعلاه.

## أسئلة شائعة (FAQ)

**س1: تلقيت بريداً إلكترونياً يطلب مني مشاركة بيانات عميل. ماذا أفعل؟**

ج1: لا ترد على البريد الإلكتروني. قم بإعادة توجيهه فوراً إلى قسم تقنية المعلومات للتحقق من صحته. نحن لا نطلب بيانات العملاء عبر البريد الإلكتروني بهذه الطريقة أبداً.

**س2: هل يمكنني استخدام كلمة المرور الخاصة بي في Beds24 على جهاز الكمبيوتر في المنزل؟**

ج2: لا، يجب استخدام بيانات تسجيل الدخول الخاصة بالعمل على أجهزة الشركة فقط ومن خلال الشبكات الآمنة المعتمدة.

**س3: أحد العملاء يطلب تفاصيل الدفع الخاصة به عبر الهاتف. هل يمكنني تزويده بها؟**

ج3: لا. لا تشارك أي تفاصيل دفع عبر الهاتف. يمكنك إرشاد العميل إلى كيفية الوصول إلى سجل مدفوعاته عبر حسابه الشخصي على المنصة.

**س4: ما هي الجهة المسؤولة عن إدارة صلاحيات الوصول في النظام؟**

ج4: المسؤول (Admin) هو المسؤول عن تحديد وتعديل صلاحيات المستخدمين المختلفين (Ops, Finance, Support) حسب مهامهم الوظيفية. راجع دليل [الأدوار والصلاحيات](../ar/01-roles-permissions.md) لمزيد من التفاصيل.
